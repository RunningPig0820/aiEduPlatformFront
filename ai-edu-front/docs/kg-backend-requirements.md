# 知识图谱后端接口需求

> 本文档列出前端知识图谱页面所需、但后端当前尚未实现的接口。
> 请后端团队根据此文档补充接口。

---

## 一、背景说明

### 知识图谱系统概述

本平台的知识图谱模块用于可视化展示 K12 教育领域的知识点体系及其关联关系。核心功能包括：

1. **教材导航树** — 用户通过层级树浏览教材体系，逐级展开到知识点
2. **知识图谱可视化** — 选中知识点后，以图的方式展示该知识点与其他知识点的前置/后续/相关关系
3. **知识点详情查看** — 点击树节点或图谱节点，查看知识点的详细信息（难度、重要性、认知层级等）
4. **同步管理** — 管理员可手动触发知识图谱数据同步（从 Neo4j → MySQL），查看同步状态和历史记录
5. **系统统计** — 展示知识图谱整体数据概览（教材数、知识点总数等）

### 数据来源

知识图谱的底层数据存储在 **Neo4j 图数据库**中，后端通过定时任务或手动触发的方式将 Neo4j 数据同步到 **MySQL 关系数据库**。前端所有请求均访问 MySQL 接口（即后端 REST API），不直接连接 Neo4j。

### 导航树的层级结构

知识图谱的导航树采用以下 6 级层级结构，从宏观到微观：

```
├── 学科（SUBJECT）          — 如：数学、语文、英语、物理、化学、生物
│   └── 年级（GRADE）        — 如：一年级、二年级、七年级、八年级
│       └── 教材（TEXTBOOK）  — 如：人教版小学数学一年级上册、北师大版初中物理八年级
│           └── 章节（CHAPTER）— 如：第一章 加法和减法
│               └── 小节（SECTION）— 如：1.1 加法的初步认识
│                   └── 知识点（POINT）— 如：加法的意义、加法的交换律
```

每个知识点是叶子节点，点击知识点后中间图谱区域展示该知识点与其他知识点的关联关系图。

## 二、新增接口需求

### 2.1 导航树层级接口（P0 - 最高优先级）

#### 需求描述

前端导航树当前只有 4 级（教材 → 章节 → 小节 → 知识点），需要扩展为 6 级（学科 → 年级 → 教材 → 章节 → 小节 → 知识点）。为此需要后端提供学科和年级维度的查询接口。

**数据来源**：这些接口从 MySQL 中的教材/知识点表中按学科、年级维度聚合查询得出。

#### 2.1.1 获取学科列表

| 项目 | 值 |
|------|-----|
| **接口路径** | `GET /api/auth/kg/subjects` |
| **说明** | 返回系统中存在的所有学科名称列表，作为导航树的根节点 |
| **认证** | 需要登录 |

**业务逻辑**：
1. 从教材表（或知识点表）中提取所有不重复的 `subject`（学科）字段值
2. 按学科名称排序返回
3. 只返回有数据（至少包含一个教材或知识点）的学科

**响应格式**：
```json
{
  "code": "00000",
  "message": "success",
  "data": ["数学", "语文", "英语", "物理", "化学", "生物"]
}
```

**前端使用方式**：
- 返回的字符串数组直接作为导航树根节点渲染
- 每个学科名称作为节点的 `uri` 和 `label`

---

#### 2.1.2 获取学科下的年级列表

| 项目 | 值 |
|------|-----|
| **接口路径** | `GET /api/auth/kg/subjects/{subject}/grades` |
| **说明** | 返回指定学科下包含的所有年级，作为导航树的第二级 |
| **认证** | 需要登录 |

**业务逻辑**：
1. 根据学科名称（如"数学"）查询该学科下所有教材
2. 从教材中提取不重复的 `grade`（年级）字段
3. 按年级排序（建议按小学→初中→高中或按年级数字排序）返回
4. 只返回有教材的年级

**请求参数**：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| subject | Path | String | 是 | 学科名称，如"数学" |

**响应格式**：
```json
{
  "code": "00000",
  "message": "success",
  "data": [
    { "uri": "一年级", "label": "一年级" },
    { "uri": "二年级", "label": "二年级" },
    { "uri": "三年级", "label": "三年级" },
    { "uri": "四年级", "label": "四年级" },
    { "uri": "五年级", "label": "五年级" },
    { "uri": "六年级", "label": "六年级" }
  ]
}
```

**字段说明**：
- `uri`：年级标识，前端用于请求下一级（教材列表）。如果年级有唯一编码请使用编码，否则直接使用年级名称
- `label`：年级的显示名称

---

#### 2.1.3 获取年级下的教材列表

| 项目 | 值 |
|------|-----|
| **接口路径** | `GET /api/auth/kg/grades/{grade}/textbooks` |
| **说明** | 返回指定年级下包含的所有教材，作为导航树的第三级 |
| **认证** | 需要登录 |

**业务逻辑**：
1. 根据年级标识（如"一年级"）查询该年级下的所有教材
2. 按教材的排序字段（orderIndex）或名称排序返回
3. 返回字段与现有 `GET /api/auth/kg/textbooks` 的 `KgTextbookDTO` 格式一致

**请求参数**：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| grade | Path | String | 是 | 年级标识，如"一年级" |

**响应格式**：
```json
{
  "code": "00000",
  "message": "success",
  "data": [
    {
      "uri": "math-primary-g1-pep-v1",
      "label": "人教版小学数学一年级上册",
      "grade": "一年级",
      "subject": "数学",
      "phase": "小学",
      "status": "ACTIVE"
    },
    {
      "uri": "math-primary-g1-bsv-v1",
      "label": "北师大版小学数学一年级上册",
      "grade": "一年级",
      "subject": "数学",
      "phase": "小学",
      "status": "ACTIVE"
    }
  ]
}
```

**字段说明**：与现有 `KgTextbookDTO` 保持一致
- `uri`：教材唯一标识，用于请求下一级（章节列表）
- `label`：教材名称
- `grade`：所属年级
- `subject`：所属学科
- `phase`：所属学段（小学/初中/高中）
- `status`：教材状态（ACTIVE/INACTIVE 等）

### 2.2 图谱关系接口（P0 - 最高优先级）

#### 需求描述

选中知识点后，中间图谱区域需要展示该知识点与其他知识点之间的关系图。前端使用 React Flow 渲染，需要后端返回节点和边的数据。

**数据来源**：Neo4j 图数据库中知识点之间的关联关系（如"前置"、"后续"、"相关"、"包含"等关系类型）。

#### 2.2.1 获取知识点的图谱关系

| 项目 | 值 |
|------|-----|
| **接口路径** | `GET /api/auth/kg/knowledge-points/{uri}/graph` |
| **说明** | 返回指定知识点及其关联的知识图谱数据，包括节点列表和边列表 |
| **认证** | 需要登录 |

**业务逻辑**：
1. 根据知识点 URI 在 Neo4j 中查询该知识点节点
2. 查询与该知识点有直接关联关系的所有知识点（如 2 度以内关联）
3. 查询知识点之间的边（关系类型）
4. 将查询结果封装为 nodes + edges 格式返回

**请求参数**：

| 参数 | 位置 | 类型 | 必填 | 说明 |
|------|------|------|------|------|
| uri | Path | String | 是 | 知识点 URI |

**响应格式**：
```json
{
  "code": "00000",
  "message": "success",
  "data": {
    "nodes": [
      {
        "id": "node-001",
        "type": "kp",
        "label": "加法的意义",
        "data": {
          "uri": "math-add-meaning",
          "difficulty": "EASY",
          "importance": "CORE"
        }
      },
      {
        "id": "node-002",
        "type": "textbook_kp",
        "label": "加法的初步认识",
        "data": {
          "uri": "math-add-intro",
          "difficulty": "EASY",
          "importance": "HIGH"
        }
      }
    ],
    "edges": [
      {
        "source": "node-001",
        "target": "node-002",
        "label": "前置"
      }
    ]
  }
}
```

**字段说明**：

`nodes[]`（节点数组）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | String | 是 | 节点唯一标识，前端渲染用。每个节点的 id 必须不同 |
| type | String | 是 | 节点类型，`kp` 表示普通知识点，`textbook_kp` 表示教材知识点 |
| label | String | 是 | 节点显示名称 |
| data | Object | 否 | 额外数据对象 |
| data.uri | String | 否 | 知识点 URI，点击节点后查看详情用 |
| data.difficulty | String | 否 | 难度等级：EASY / MEDIUM / HARD |
| data.importance | String | 否 | 重要性等级：LOW / MEDIUM / HIGH / CORE |

`edges[]`（边数组）：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| source | String | 是 | 源节点 id（对应 nodes 中的 id） |
| target | String | 是 | 目标节点 id（对应 nodes 中的 id） |
| label | String | 否 | 边的关系类型，如"前置"、"后续"、"相关"、"包含" |

**前端渲染逻辑**：
- 根据节点的 `type` 字段选择不同样式的节点（教材知识点用蓝色主题，普通知识点用灰色主题）
- 根据边的 `label` 字段在边中间显示关系文字
- 前端使用 React Flow 的力导向布局自动计算节点位置
- 如果后端能直接返回节点坐标（x, y），前端优先使用后端坐标

**注意事项**：
- 如果知识点没有关联关系，返回空数组 `{"nodes": [], "edges": []}`
- 建议返回的节点数量不超过 50 个，避免前端渲染性能问题
- 选中知识点本身也应该包含在 nodes 数组中

## 三、同步管理接口确认

### 3.1 已有同步接口

以下接口后端已实现，前端已对接：

| 接口 | 方法 | 说明 | 状态 |
|------|------|------|------|
| `/api/auth/kg/sync/full` | POST | 全量同步知识图谱数据 | 已实现 ✓ |
| `/api/auth/kg/sync/status` | GET | 获取同步状态 | 已实现 ✓ |
| `/api/auth/kg/sync/records` | GET | 获取同步记录列表 | 已实现 ✓ |

### 3.2 同步参数说明

`POST /api/auth/kg/sync/full` 的请求参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| subject | String | 否 | 学科名称，如"数学"。不传则同步所有学科 |
| phase | String | 否 | 学段，如"小学"、"初中"、"高中" |
| grade | String | 否 | 年级，如"一年级" |
| textbookUri | String | 否 | 教材 URI，指定单本教材同步 |

**业务逻辑**：
- 如果所有参数都不传，执行全量同步（所有学科、所有年级、所有教材）
- 如果传了部分参数，只同步符合条件的数据
- 参数可以组合使用，如 `subject=数学&grade=一年级` 表示只同步数学一年级的数据
- 同步是一个异步操作，接口立即返回同步任务信息，前端通过 `/sync/status` 查询进度

## 六、前端使用场景说明

### 场景 1：用户浏览知识图谱

```
1. 用户进入知识图谱页面
2. 前端调用 GET /subjects → 显示学科列表（数学、语文、英语...）
3. 用户点击"数学" → 前端调用 GET /subjects/数学/grades → 显示年级列表
4. 用户点击"一年级" → 前端调用 GET /grades/一年级/textbooks → 显示教材列表
5. 用户点击"人教版一年级上册" → 前端调用 GET /textbooks/{uri}/chapters → 展开章节
6. 逐级展开章节 → 小节 → 知识点
7. 用户点击某个知识点 → 前端调用 GET /knowledge-points/{uri}/graph → 展示图谱关系
```

### 场景 2：管理员手动同步知识图谱

```
1. 管理员点击页面右上角"同步"按钮
2. 弹出同步对话框，包含三个下拉选择器：
   - 教材（第一位置，可选）：下拉选项来自 GET /textbooks
   - 学科（可选）：下拉选项来自 GET /subjects
   - 年级（可选）：下拉选项来自前端本地维护
3. 管理员选择筛选条件后点击"开始同步"
4. 前端调用 POST /sync/full，携带选中的筛选参数
5. 同步完成后，前端自动刷新同步状态面板
```

### 场景 3：系统统计概览

```
1. 页面加载时，前端调用 GET /system/stats → 展示教材数、章节数、知识点总数
2. 前端调用 GET /neo4j/health → 展示 Neo4j 连接状态
```
