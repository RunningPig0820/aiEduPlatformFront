## ADDED Requirements

### Requirement: 用户应能在右侧面板查看选中节点的详情信息

系统应在右侧面板中展示被选中节点的基本信息，包括节点名称、学科、年级、难度、认知层级、URI 等。

#### Scenario: 选中教材知识点节点
- **WHEN** 用户在左侧树或中间图谱中点击教材知识点节点
- **THEN** 右侧面板展示该教材知识点的名称、学科、年级、单元、课时、URI 等信息

#### Scenario: 选中知识点节点
- **WHEN** 用户在左侧树或中间图谱中点击知识点节点
- **THEN** 右侧面板展示该知识点的名称、学科、难度、认知层级、URI 等信息

#### Scenario: 未选中任何节点时
- **WHEN** 用户尚未选择任何节点
- **THEN** 右侧面板显示占位提示文本（如「请选择一个节点查看详情」）

### Requirement: 详情面板应区分节点类型

不同类型节点（教材知识点 / 知识点）的详情面板展示不同字段。

#### Scenario: 教材知识点详情面板
- **WHEN** 选中教材知识点类型节点
- **THEN** 面板标题为「教材知识点详情」，展示字段包括：名称、学科、年级、单元、课时、URI

#### Scenario: 知识点详情面板
- **WHEN** 选中知识点类型节点
- **THEN** 面板标题为「知识点详情」，展示字段包括：名称、学科、难度、认知层级、URI

### Requirement: 详情面板应支持复制 URI 操作

用户可以一键复制节点的 URI 到剪贴板。

#### Scenario: 复制 URI 成功
- **WHEN** 用户点击 URI 旁的复制按钮
- **THEN** URI 复制到剪贴板，显示「已复制」提示
