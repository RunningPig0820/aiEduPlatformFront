## ADDED Requirements

### Requirement: 用户应能通过树形导航浏览教材知识点层级

系统应提供逐级展开的树形导航组件，展示 教材 → 学科 → 年级 → 单元 → 课时 → 知识点 的层级结构。

#### Scenario: 页面加载时显示教材根节点
- **WHEN** 用户打开知识图谱页面
- **THEN** 左侧显示教材根节点列表（如人教版），初始仅展开第一层

#### Scenario: 用户展开树节点时懒加载子节点
- **WHEN** 用户点击展开某个树节点
- **THEN** 系统请求该节点的子节点列表并显示，已展开的节点缓存子节点数据

#### Scenario: 用户逐级展开到知识点层级
- **WHEN** 用户依次展开 教材 → 学科 → 年级 → 单元 → 课时
- **THEN** 系统显示该课时下的知识点列表

### Requirement: 树节点应区分节点类型并展示不同图标

不同类型的树节点（教材/学科/年级/单元/课时/知识点）应使用不同的图标或样式区分。

#### Scenario: 知识点节点使用特殊标识
- **WHEN** 树节点为知识点类型
- **THEN** 节点前显示 ● 标识，区别于父级节点（章节/单元等使用 ▼ 标识）

### Requirement: 点击知识点节点应触发关联事件

用户点击树中的知识点节点时，应通知中间图谱区域和右侧详情面板。

#### Scenario: 用户点击树中知识点节点
- **WHEN** 用户点击树中的知识点类型节点
- **THEN** 中间区域加载该知识点的关系图谱，右侧面板显示该节点的详情
